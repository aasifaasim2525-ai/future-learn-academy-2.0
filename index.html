Module 3 – Variables & Data Types

Why this matters: Variables are named storage boxes. They let you store values, reuse them, and change them. Python figures out types automatically (dynamic typing). Common types: int, float, str, bool. Example code: 
age = 25
price = 19.99
name = 'Alice'
is_student = True
print(type(age), type(price), type(name), type(is_student))
Expected Output: 
<class 'int'> <class 'float'> <class 'str'> <class 'bool'>
Challenge: Create variables of int, float, and string types. Convert a numeric string into a number and perform a calculation.

Module 4 – String Operations & Formatting

Why this matters: Strings are text. You will use them for outputs, user input, and files. Formatting strings with f-strings makes combining text and variables easy and readable. Lesson: Use concatenation, methods like .upper() and .replace(), and f-strings to embed values. Example code: 
greet = 'Hello'
name = 'Bob'
msg = f"{greet}, {name}!"
print(msg)
print(msg.upper())
print(msg.replace('Bob','Alice'))
age = 30
print(f"{name} is {age} years old.")
Expected Output: 
Hello, Bob!
HELLO, BOB!
Hello, Alice!
Bob is 30 years old.
Challenge: Simulate asking a user for name and age (via comments) and print a greeting using an f-string and two string methods.

Module 6 – Lists

Why this matters: Lists store ordered collections of items you can change. They are ideal for sequences like student names, scores, or any group of values. Lesson: Use indexing, slicing, and methods like append(), remove(), pop(), and sort(). Example code: 
nums = [10, 20, 30, 40]
print(nums[0], nums[-1], nums[1:3])
nums.append(50)
nums.remove(20)
print(nums)
nums.sort(reverse=True)
print(nums)
Expected Output: 
10 40 [20, 30]
[10, 30, 40, 50]
[50, 40, 30, 10]
Challenge: Create a list of 5 items, change one, slice a subset, and use two list methods.

Module 7 – Tuples & Sets

Why this matters: Tuples are immutable sequences—use them for fixed data. Sets store unique items and are great for removing duplicates or checking membership quickly. Example code: 
coords = (1, 2)
print(coords, type(coords))
s = {1, 2, 2, 3}
print(s)
s.add(4)
s.discard(2)
print(s)
Expected Output: 
(1, 2) <class 'tuple'>
{1, 2, 3}
{1, 3, 4}
Challenge: Define a tuple and a set. Attempt to modify both and observe the results.

Module 8 – Dictionaries

Why this matters: Dictionaries store key→value pairs. They are perfect for structured data like profiles, settings, or lookup tables where you want to retrieve by name. Lesson: Access with keys, change values, and iterate with .items(). Example code: 
person = {'name': 'Alice', 'age': 25}
print(person['name'])
person['age'] = 26
person['city'] = 'London'
for key, val in person.items():
    print(key, '->', val)
Expected Output: 
Alice
name -> Alice
age -> 26
city -> London
Challenge: Make a 3-item dictionary, update one value, and loop through keys and values.

Module 9 – Control Flow: if, elif, else

Why this matters: If statements let your code make decisions based on data. Use elif for multiple branches and else as a fallback. Example code: 
num = 5

    print('Positive')
elif num == 0:
    print('Zero')
else:
    print('Negative')

Positive

Module 10 – Loops: for & while

Why this matters: Loops repeat work so you don't need to write the same code many times. Use for to iterate sequences and range(); while repeats until a condition changes. Example code: 
for i in range(3):
    print('Loop', i)

n = 0
while n < 3:
    print('While', n)
    n += 1
Expected Output: 
Loop 0
Loop 1
Loop 2
While 0
While 1
While 2
Challenge: Use both loops to print even numbers from 0 to 10.

Module 11 – Loop Control: break, continue, else

Why this matters: break stops a loop early, continue skips the rest of the current loop iteration, and else on a loop runs if the loop did not hit break. Example code: 
for i in range(5):
    if i == 3:
        break
    print(i)
else:
    print('Completed loop')

for i in range(5):
    if i % 2 == 0:
        continue
    print(i)
Expected Output: 
0
1
2
1
3
Challenge: Find the first multiple of 7 between 1 and 100 using a loop and break.

Module 13 – Scope & Lifetime (Beginner Friendly)

Why this matters: Scope explains which variable you're referring to when you use a name. Understanding scope helps you avoid confusing bugs like trying to use a variable before it exists. Key idea — LEGB: Local → Enclosing → Global → Built-in. Example code: 
x = 10  # global

def show():
    x = 99
    print('inside:', x)

show()
print('outside:', x)
Expected Output: 
inside: 99
outside: 10
Challenge: Write a function that counts how many times it was called using nonlocal. Avoid using names like 'list' or 'sum' for variables.

Module 14 – Error Handling: try/except/finally

Why this matters: Errors happen. Use try/except to catch errors and keep your program running. Use finally to run cleanup actions that should always happen. Example code: 
try:
    result = 10 / 0
except ZeroDivisionError:
    print('Cannot divide by zero!')
finally:
    print('Done.')
Expected Output: 
Cannot divide by zero!
Done.
Challenge: Ask user for a number and safely convert it to int. If conversion fails, ask again.

Module 18 – Classes & Objects

Why this matters: OOP models real things as objects with attributes and behavior. Classes help organize related data and functions into neat packages. Example code: 
class Person:
    def __init__(self, name, age):
        self.name = name
        self.age = age
    def greet(self):
        return f"Hi, I'm {self.name} and I'm {self.age}."

p = Person('Alice', 30)
print(p.greet())
Expected Output: 
Hi, I'm Alice and I'm 30.

Module 19 – Simple OOP Project: Contact Manager

Why this matters: Building a small app shows how classes and lists/dictionaries work together. A Contact class organizes contact data; a list stores many contacts. Example code: 
class Contact:
    def __init__(self, name, phone):
        self.name = name
        self.phone = phone
    def __str__(self):
        return f"{self.name}: {self.phone}"

contacts = [Contact('Bob', '1234'), Contact('Sue', '5678')]
for c in contacts:
    print(c)
Expected Output: 
Bob: 1234
Sue: 5678
Challenge: Add functions to add, remove, and search contacts; save/load to a file.
